# Lupi Molecule Reliability and Nomenclature

This is the curation contract for gallery molecules and materials that Lupi is
allowed to present as selected, source-backed examples.

## Goals

- Keep the visual asset loadable and reproducible.
- Keep identity separate from the display label.
- Keep coordinates, bonds, names, formulas, and properties tied to explicit
  source evidence.
- Make selected molecules searchable by common name, systematic name, formula,
  PubChem CID, and aliases.

## Naming Fields

Use these fields for any selected molecule or material that graduates from a
demo asset into the curated gallery.

| Field | Meaning |
| --- | --- |
| `title` | Human-facing short label in `gallery-data.json`. Prefer common or teaching names. |
| `preferredName` | Stable identity label in `gallery-nomenclature.json`. |
| `systematicName` | IUPAC/systematic name when available and useful. |
| `molecularFormula` | Source-backed formula for molecules; composition/reduced formula for materials. |
| `aliases` | Searchable common names, abbreviations, legacy names, and teaching names. |
| `sourceUrl` | Exact source page or durable source record. |
| `geometrySource` | Where coordinates came from, not what they visually resemble. |
| `confidence` | `source-backed`, `computed`, `procedural`, or `illustrative`. |

Keep abbreviations as aliases unless the abbreviation is the best public label.
For example, the gallery may show `THC`, but the reliability catalog should also
carry `Tetrahydrocannabinol`, `delta-9-THC`, the PubChem CID, and formula.

## Reliability Tiers

1. `source-backed`: identity and coordinates come from a known database,
   publication artifact, or standards source. Examples: PubChem 3D conformer,
   PDB, COD, Materials Project, NIST IPR, OMol25.
2. `computed`: generated by a named local workflow with method, model/potential,
   version, seed or script path, and enough metadata to rerun.
3. `procedural`: constructed by a deterministic builder for teaching or viewer
   stress testing. The structure is useful, but it is not experimental truth.
4. `illustrative`: visual or artistic atom layouts such as QR/point-art assets.
   These must never be described as physical molecules.

## Promotion Checklist

Before a molecule becomes a selected Lupi example:

1. The asset exists locally or has a trusted, CORS-safe remote URL.
2. A manifest run records file size and SHA-256 for local assets.
3. PubChem/PDB/COD/Materials Project/NIST/OMol identifiers are present when the
   source is one of those databases.
4. Formula or composition agrees with the shipped coordinates when coordinates
   are small enough to inspect.
5. Bonds are labeled correctly: source topology if supplied, visual guides if
   inferred by the viewer.
6. `subtitle` explains why the structure is useful without overstating source
   truth.
7. Abbreviations have aliases and expanded names in the nomenclature catalog.

## Commands

Run the local audit without writing artifacts:

```bash
pnpm audit:nomenclature
```

Write a reproducibility manifest and local asset backup under ignored
`.verify-artifacts/gallery-reliability/`:

```bash
pnpm backup:gallery
```

Run the focused gallery guard after metadata changes:

```bash
pnpm --filter @atlas/ui test -- src/gallery-data.test.ts
```
