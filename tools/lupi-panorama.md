# Lupi Environment Pack Workflow

Lupi backgrounds are now treated as authored equirectangular environment media,
not as ad-hoc JPEG replacements. The viewer runtime supports a single media
contract for gradients, still panoramas, and future panoramic video loops.

## Runtime Contract

Environment presets live in `packages/ui/src/backgroundPresets.ts`.

Each preset resolves to one of:

- `gradient`: generated 2:1 equirectangular canvas fallback.
- `image`: still 2:1 equirectangular panorama.
- `video`: muted looping equirectangular panorama video with a poster image.

The scene background and XR dome both consume the same media texture hook, so
desktop, VR, and AR all share projection, wrap, color-space, and cleanup rules.

  --negative-suffix "repeating tile, mirrored quadrants"

The generated Comfy workflow for each asset is saved beside the candidate image.
The current static flow is:

1. Z-Image Turbo GGUF base render at `1024x512`.
2. `VAEDecode` or optional `VAEDecodeTiled`.
3. Optional WAS `Image Seamless Texture` repair.
4. `4x-UltraSharp.pth` upscaling.
5. Lanczos normalize to the target 2:1 size.
6. JPEG candidate write plus QA metrics.

## Publication Contexts

Practical figure backgrounds live in `tools/lupi-publication-backgrounds.json`
and are generated by `tools/lupi_publication_backgrounds.py`. These are not
fantasy scenes; they are restrained context plates for publication and review
figures:

- `Figure Neutral`: clean print/export plate for methods and journal snapshots.
- `Cryo-EM Grid`: support-film context for biomolecular structures.
- `Diffraction Plate`: crystallography / diffraction context for lattice views.
- `Density Contours`: electron-density or potential-map context for fitted geometry.
- `Spectrum Strip`: Raman/IR-style spectrum context for spectroscopy narratives.
- `Lab Notebook`: light teaching/review background with grid-note structure.
- `Beamline Slate`: instrument/beamline context for structure determination.
- `Phase Map`: materials phase/thermo context for composition or defect studies.

Regenerate them with:

```powershell
python .\tools\lupi_publication_backgrounds.py --all
```

## Panoramic Video Direction

For motion backgrounds, keep the still panorama as the poster/source master and
generate short, subtle loops from it. The authored motion contract lives in
`tools/lupi-motion-loops.json`; both the encoder and the viewer preset registry
read that manifest so labels, output names, tiers, and motion recipes stay in
one place.

Prefer:

- Tiered runtime video variants. The current pack emits `quality` (`2048x1024`)
  and `preview` (`1024x512`).
- 8-15 second muted loops at 24 fps.
- MP4 for the first deployable pass; add WebM variants when target coverage
  justifies the extra assets.
- Calm center composition so molecular geometry and HUD text stay readable.
- QA on first/last-frame loop delta, left/right seam delta across sampled frames,
  browser decode, and desktop/mobile canvas screenshots.

Many Lupi environments should be procedural or layer-animated instead of fully
AI-video generated: star drift, phosphor scanlines, circuit pulses, aurora
shimmer, smoke, and particulate fields will loop more cleanly and cost less.

Generate the current motion pack from accepted still masters:

```powershell
python .\tools\lupi_motion_loops.py --all --tier quality
python .\tools\lupi_motion_loops.py --all --tier preview
```

The generated files are authored app assets under `apps/web/public/backgrounds`.
They are intentionally compatible with the same `video` media contract that a
future local Comfy image-to-video workflow should write into.

The encoder streams RGB frames directly to ffmpeg, writes local cache records
under `tools/lupi-motion-cache/`, and skips unchanged recipe/tier/source
combinations unless `--force` is passed. Use `--all-tiers` to regenerate every
manifest tier and `--jobs 2` or higher for parallel batch encoding.

Run browser QA for the deployed motion presets:

```powershell
node .\tools\lupi_video_qa.mjs --url http://127.0.0.1:3000
node .\tools\lupi_video_qa.mjs --url http://127.0.0.1:3000 --tier preview --preset quantum-fog-loop
```

The QA script opens the built viewer with each loop selected, verifies the MP4
is requested for the selected manifest tier, records hidden video playback
state, and saves settled/later screenshots plus `report.json` under
`.verify-artifacts/lupi-video-qa`.
